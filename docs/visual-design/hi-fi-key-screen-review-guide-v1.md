## 2. What This Deck Covers

The HTML deck contains the first six hi-fi key-screen comps:

1. Home
2. Match Flow
3. Results
4. Tutor Profile
5. Booking
6. Tutor Overview

These are the first screens that most strongly define:

- the matching-first shift
- the non-generic visual direction
- the shared student and tutor ecosystem

## 3. Core Direction Used

This deck follows:

- `Route A: Editorial Guidance` as the main visual route
- `Route B` only as a structural influence for tutor operational density

That means the comps intentionally use:

- warm paper surfaces
- dark ink typography
- forest as the primary action color
- clay only for selective emphasis
- stacked decision rows instead of marketplace grids
- one visible component language across student and tutor screens

## 4. What To Review First

If you review only a few things first, focus on these questions:

1. Does the Home screen feel distinct and memorable instead of generic edtech?
2. Does Match Flow feel psychologically easy and guided?
3. Do Results feel like ranked decisions rather than inventory browsing?
4. Does Tutor Profile put fit and trust ahead of biography?
5. Does Booking feel calm and clear around schedule and policy?
6. Does Tutor Overview still feel like the same product as the student side?

## 5. Screen-Specific Review Focus

### Home

Review:

- hero presence
- problem-chip usefulness
- whether the sample match story feels stronger than a normal marketplace hero

### Match Flow

Review:

- question hierarchy
- summary-bar visibility
- whether the helper rail clarifies without becoming noise

### Results

Review:

- readability of `MatchRow`
- fit statement hierarchy
- shortlist tray usefulness

### Tutor Profile

Review:

- first-viewport confidence
- balance between proof and biography
- booking rail clarity

### Booking

Review:

- schedule surface clarity
- timezone visibility
- whether the summary rail reduces uncertainty

### Tutor Overview

Review:

- whether it feels action-led instead of KPI-led
- whether it still feels visually related to the student-facing screens
- whether the panel density is right

## 6. Expected Outcome

After reviewing the hi-fi deck, we should be able to decide:

- what visual route to keep without hesitation
- which screen sections need a second pass
- whether the density balance between student and tutor is right
- whether we are ready to expand hi-fi into the next screens
