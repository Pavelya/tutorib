## 2. What This Deck Covers

The second HTML deck covers:

1. Compare
2. Messages
3. Tutor Lessons
4. Tutor Schedule
5. Tutor Students
6. Tutor Application

These screens matter because they prove whether the shared ecosystem is real after the first conversion-focused screens.

## 3. Core Direction Used

This deck continues the same approved logic:

- `Route A: Editorial Guidance` as the primary visual route
- `Route B` only as structural discipline for denser tutor workflows

That means the screen family should still feel:

- warm
- specific
- calm
- readable
- operational without becoming dashboard-generic

## 4. What To Review First

If you review only a few things first, focus on these questions:

1. Does Compare feel faster and clearer than opening many tutor profiles?
2. Do Messages feel like a lesson-aware teaching shell instead of a generic chat tool?
3. Do Tutor Lessons still feel like the same lesson ecosystem as the student side?
4. Does Tutor Schedule make editing and student-visible outcome feel connected?
5. Does Tutor Students feel like a teaching relationship view rather than a CRM?
6. Does Tutor Application feel staged and encouraging instead of bureaucratic?

## 5. Screen-Specific Review Focus

### Compare

Review:

- readability of the table
- whether differences in fit, trust, and availability are obvious quickly
- whether the bottom action area helps the decision

### Messages

Review:

- strength of the thread header
- clarity of lesson context
- whether the conversation shell feels shared across roles

### Tutor Lessons

Review:

- card readability
- lesson detail usefulness
- whether tutor actions feel layered onto the same shared lesson object

### Tutor Schedule

Review:

- clarity of the schedule canvas
- quality of the preview relationship
- whether the rules and notice logic feel integrated instead of scattered

### Tutor Students

Review:

- strength of the identity-first layout
- usefulness of the student detail preview
- whether the page avoids generic CRM energy

### Tutor Application

Review:

- confidence level of the flow
- balance between main task and helper rail
- whether completion guidance feels helpful instead of punitive
